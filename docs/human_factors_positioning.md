# Human Factors Positioning

This demo connects to human factors and ergonomics through cognitive and affective ergonomics rather than physical posture-risk assessment. It uses wearable physiological and motion signals to estimate human state under different protocol conditions. In this framing, the target is not body posture, joint load, or manual handling risk. The target is psychophysiological strain: whether signals collected from the wrist contain enough information to distinguish a lower-strain baseline condition from an elevated stress condition.

WESAD is suitable for a first cognitive ergonomics demo because it provides synchronized wearable sensing data collected under controlled affective conditions. The Empatica E4 signals include electrodermal activity, blood volume pulse, temperature, and acceleration, which are commonly used in stress and affective computing studies. The dataset also includes protocol metadata, making it possible to connect signal windows with experimental conditions in a transparent way. This makes WESAD a useful starting point for demonstrating how human-state assessment can be built from raw wearable data.

The current pipeline demonstrates a complete, reproducible baseline workflow. It reads raw Empatica E4 CSV files, uses protocol timing to assign labels, segments signals into fixed-length windows, extracts simple statistical features, trains baseline classifiers for baseline versus stress recognition, and evaluates generalization with leave-one-subject-out validation. This shows several technical abilities relevant to human factors research: raw wearable signal processing, protocol-based labeling, feature extraction, state classification, and subject-aware evaluation.

The demo also has clear boundaries. It does not cover posture assessment, RULA or REBA scoring, musculoskeletal load estimation, physical fatigue modeling, or real-time deployment. It should not be interpreted as a finished stress-monitoring product or as a validated occupational health system. The current results are best read as an interpretable baseline for wearable human-state sensing.

Future extensions could make the work more relevant to cognitive ergonomics and decision-oriented human factors applications. Useful additions include questionnaire analysis, subject-level reliability assessment, workload index construction, advanced EDA and BVP features, and intervention decision support. These extensions would move the demo from basic condition recognition toward systems that can evaluate confidence, track individual differences, and support human-centered operational decisions.
