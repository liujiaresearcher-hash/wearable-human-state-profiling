# Research Positioning

This demo is relevant to my previous background because it uses a familiar assessment logic in a new sensing domain. My earlier work focused on infrastructure sensing, condition assessment, quality evaluation, and decision-oriented analysis. In those projects, raw monitoring data were not treated as an end in themselves. They were processed, checked, transformed into interpretable indicators, and used to support assessment and decisions about system state.

The same structure appears in this wearable sensing demo. Instead of assessing the state of infrastructure components or construction quality, the pipeline assesses human state from physiological and motion signals. The objects are different, but the technical reasoning is similar: raw sensor data are converted into structured evidence, evidence is summarized through features, and features are evaluated against a state-assessment task.

The current task is baseline versus stress recognition using raw Empatica E4 signals from the WESAD dataset. The pipeline follows a transparent chain: raw wearable sensor data, preprocessing and protocol alignment, window segmentation, feature engineering, state classification, and evaluation. This makes the demo a bridge between infrastructure monitoring and human factors research. It shows how methods used for engineered-system assessment can be transferred toward human-centered sensing, affective computing, and embodied interaction analysis.

Technically, the demo demonstrates raw data processing, signal segmentation, statistical feature engineering, supervised state classification, cross-subject evaluation, and reproducible research practice. The leave-one-subject-out setting is especially important because it moves beyond within-subject fitting and asks whether simple wearable features generalize across people.

This repository should be understood as a baseline research demo, not a final deployable stress-recognition system, clinical diagnostic method, or occupational health product. It is designed to be readable, reproducible, and extensible. Future extensions could better reflect my strengths by adding data quality assessment for wearable signals, subject-level reliability analysis, and multi-modal fusion across physiological and behavioral channels. These directions would connect the demo more directly to decision-oriented research assessment: not only classifying a protocol condition, but also studying confidence, reliability, and reproducibility.
